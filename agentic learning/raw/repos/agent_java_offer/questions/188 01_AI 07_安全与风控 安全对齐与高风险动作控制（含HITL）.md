## 题目正文

### 1. 子问题：安全对齐与高风险动作控制（含HITL）

主问题：如何确保 Agent 安全、可控、可追责？

口述答案：
安全要分层做：模型层做对齐训练，系统层做最小权限与工具白名单，执行层做沙箱隔离与资源限额，业务层对高风险动作强制人工审批。再配合红队测试、注入攻击测试、全链路审计日志和告警阈值。核心原则是“先可控再智能”：即使模型偶发漂移，也不能直接触达不可逆动作。

常见追问：

1. 如何设计高风险动作审批流？
2. 红队测试怎么选样本与评估口径？
3. 安全与体验冲突时如何平衡？
