## 题目正文

### 如何设计高并发的朋友圈点赞系统架构？

点赞和评论通常是独立数据，不建议和主帖子强耦合。

常见做法：
1. **Redis 存点赞集合**
- 每条朋友圈对应一个 Redis Set；
- Set 里放给这条朋友圈点赞的用户 ID；
- 点赞数量可以直接用 `SCARD`，点赞列表可用 `SMEMBERS` 或分页拉取。

2. **好友可见点赞**
- 如果“只有好友才能看见某个点赞”，可以把“当前用户的好友集合”和“这条朋友圈的点赞用户集合”做交集判断；
- 工程上可以通过 `SISMEMBER` 之类方式快速判断是否可见。

3. **重复点赞幂等**
- 如果用数据库存点赞关系，可以用 `(user_id, post_id)` 联合唯一键保证一个人只能点一次；
- 如果用 Redis Set，本身天然去重，重复添加不会产生第二条记录。

面试里可以总结为：
**朋友圈主链路是“发帖异步分发到时间线”，点赞链路是“独立存储、缓存优先、关系裁剪可见性、幂等防重”。**
