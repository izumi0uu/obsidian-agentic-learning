## 题目正文

### 6. 问：RPC 调用失败的排查顺序是什么？

答：先看调用维度监控：超时率、错误码、重试率；再分层排查 DNS/注册发现、网络连接池、线程池和下游依赖。最后结合 TraceId 还原单次请求路径，区分“流量突增、代码回归、下游抖动”三类根因。  
追问：同样是超时，如何快速判断是网络抖动还是线程池打满？

高频连问路径：

1. 选型对比 -> 链路细节 -> 瓶颈定位
2. 注册发现 -> 心跳机制 -> 故障降级
3. 超时重试 -> 重试风暴 -> 熔断恢复
