## 题目正文

### 3. 消息积压怎么处理？

先定位根因: 判断是生产激增还是消费变慢，再看是否有慢分区或下游阻塞。
先止血: 处理上用扩容消费者、按优先级分流、临时降级非核心业务.
再做回补，避免二次冲击: 积压清理不能“满速回放”，要限速回补。单独开回补 consumer group，配合令牌桶限速，保护 DB 和下游接口。
最后补齐监控和容量治理，确保同类问题不再反复出现
