## 题目正文

### 1. CAP 在后端怎么落地理解？

你这句总结很对。CAP 真正在后端落地时，要抓住一个关键点：

1. CAP只在“发生网络分区(P)”时才要求取舍  

平时网络正常时，很多系统既能高可用也能强一致。  

一旦分区发生，你必须选：  

- 要 `C`：宁可拒绝部分请求，也不返回旧数据  

- 要 `A`：宁可先返回，后续再收敛一致

2. 后端落地不是“系统级二选一”，而是“业务分级”  

同一个公司里通常同时有 CP 和 AP 服务。  

- 交易/资金/库存扣减：偏 `CP`（错一笔代价太高）  

- 内容流/推荐/点赞计数：偏 `AP`（允许短暂不一致）  

- 用户资料展示：常做折中（读AP、关键写CP）
