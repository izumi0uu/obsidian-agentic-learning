## 题目正文

### 7. 子问题：你了解哪些专门用于评估 Agent 能力的基准测试？这些基准通常如何构建测试环境和任务？

答：
常见有 WebArena、AgentBench、GAIA。它们一般构建可复现沙箱环境（网页、终端、数据库、工具接口），给高层任务目标，让 Agent 自主拆解执行，最后通过脚本检查终态是否达成目标。相比静态题库，这类基准更贴近真实任务流程，能评估规划、工具使用和鲁棒性，而不仅是最终文本质量。

随着Agent研究的兴起，一系列专门用于评估Agent能力的基准测试被开发出来，它们的核心特点是提供**可控的、可复现的交互式环境**。

**几个知名的Agent能力基准测试：**

1. **WebArena:**
  - **专注领域：** **网页浏览与操作**。是构建了一个高度逼真的独立的网页环境磨难器，环境模拟器复刻了包括多个真实网站，如电商论坛、开发写作工具的功能，让 A 智能在这个独立的环境里完成复杂的任务。比如说是在电商网站上找到一个满足满足要求的商品并加入购物车。然后评估方式是最终在网页状态会会程序化的判断，比如说判断购物车里是否有正确的商品。
2. **AgentBench:**
  - **专注领域：** **通用Agent能力的综合评估**。Agent Bench 是一个全面，比较全面的包含了八种不同的环境来评估 Agent 的能力。比如说是操作系统环境，在 Linux 终端操作文件，执行命令。比如说在数据库环境，对一个 SQL 数据库进行查询。比如说知识图谱环境，在知识图谱中进行多条推理。比如说一些简单的文字与与冒险游戏。
3. **GAIA (General AI Assistants):**
  - **专注领域：** **模拟人类使用真实工具完成复杂任务**。就是完成，模拟人类使用真实工具完成一个比较极具挑战性的任务。一般这种任务需要多种工具并用才能解决，比如说用网页搜索、代码、文件操作才能一并解决。”

**这些基准通常如何构建测试环境和任务？**

测试环境的构建一般是构建一个可控隔离的沙箱环境, 不会让 A 生态来访问真实的互联网环境。一般是用 Docker 容器来封装一个包含浏览器终端文件系统的独立环境。对于网页浏览，本地会托管一个网站的静态版本，使用或使用外部后台来模拟响应 Agent 的请求。评估任务的话，通常会以一个高层次的目标的形式给出，不是给具体的操作步骤和命令。这样可以涵盖要测试的 Agent 的综合的能力，比如说信息检索工具的使用、推理规划和记忆等。这个任务通常也会附带一个明确的可程序化验证的成功标准。评估的话，就是你根据上面的设置一个评估标准，去整一个脚本，去来自动的检查这个沙箱环境的一些最终的任务状态。
