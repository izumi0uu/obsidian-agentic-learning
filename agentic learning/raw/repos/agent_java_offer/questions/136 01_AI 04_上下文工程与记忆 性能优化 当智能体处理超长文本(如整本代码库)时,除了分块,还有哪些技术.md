## 题目正文

### 34.性能优化:当智能体处理超长文本(如整本代码库)时,除了分块,还有哪些技术

核心思路是：不把整库塞进上下文，而是“索引化 + 检索化 + 压缩化 + 增量化”。

除了分块，还可以用这些技术：

代码图索引：用 AST、依赖图、调用图做“代码图检索”，比纯文本更准。
[[Hybrid Search|混合检索]]：[[BM25]]（关键词）+ [[Retriever|向量检索]]（语义）+ 图检索（关系）联合召回。
重排与预算控制：召回后 [[Reranking|rerank]]，只保留最相关片段；按 [[Token|token]] 预算动态调 TopK。
层级摘要：文件摘要、模块摘要、仓库摘要三级压缩；先读摘要再按需下钻原文。
增量更新：只重建变更文件（基于 git diff），避免每次全量 embedding。

处理超长代码库，我会做“代码图索引 + 混合检索 + 分层摘要 + 增量更新 ”
