## 题目正文

### 7. 端到端不丢消息怎么做？

生产端要确保发送成功确认(Kafka 常配 acks=all、enable.idempotence=true,生产端的幂等允许重复发送)，Broker 侧做持久化和多副本同步；
消费端严格执行“处理成功再提交ack位点”。同时配合幂等与消费失败补偿机制(消费失败走重试队列或延迟重试，超过阈值进 DLQ，不能无限阻塞主链路)，才能在重试场景下实现“逻辑不丢不重”。
