### 3. 子问题：你用过哪些 [[Agent Framework|Agent 框架]]？选型是如何选的？你最终场景的评价指标是什么？

答：
我最常用的主要有两个：**LangChain** 和 **LlamaIndex**。

选型：

**1.当项目是“逻辑编排驱动”时，我首选LangChain,** 这类项目的核心是构建一个复杂的、需要执行一系列步骤、并与多种外部工具（APIs, 数据库, 文件系统）交互的[[Agent]], 一个自动化的研究助手，需要先上网搜索，然后对结果进行总结，再用代码执行器进行数据分析

**2.当项目是“数据驱动”时，我首选LlamaIndex,** 这类项目的核心是构建一个围绕特定知识库的问答或分析系统，即高级[[RAG]]（Retrieval-Augmented Generation）。例如，一个能回答公司内部上千份PDF技术文档的客服机器人

**最终场景的评价指标是什么？**

**1.[[Task Success Rate|任务成功率]]**

**2.完成任务过程中的资源消耗: token费用, api费用, 延迟, 步骤数(次数越少意味[[Planning|规划]]能力越强)**

**3.鲁棒性与可预测性: 错误处理能力; 输出一致性; 安全[[Evaluation|评估]]**
