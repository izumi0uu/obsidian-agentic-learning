## 题目正文

### 8. 子问题：在[[Evaluation|评估]]一个 [[Agent]] 的任务完成情况时，除了最终结果的正确性，还有哪些过程指标是值得关注的？（例如：效率、成本、鲁棒性）

答：
过程指标建议至少看六项：步骤数、总时延、Token/费用、[[Tool Calling|工具调用]][[Task Success Rate|成功率]]、重试率、异常恢复时长。原因是很多系统“能做对但代价太高”，线上不可用。这些过程指标可直接指导优化方向，比如减少无效循环、提升[[Reranking|重排]]质量或改进工具参数校验，比只看准确率更有工程价值。

**值得关注的关键过程指标包括：**

**1. 效率 (Efficiency):**

- 成本方向，你要考虑 token 的调用量以及工具 API 的调用费用。
- 步骤数，就是 Agent 在执行、思考、行动的循环总次数。通常更少的步骤完成任务被 Agent 认为[[Planning|规划]]能力更强。
- 计算的耗时，总耗时和计算的时间。

**2. 鲁棒性 (Robustness):**

- 错误处理能力, 就是当工具类返回错误或者网页加载失败，或者遇到其他预预期之外的情况时，A 智能能否识别并采取纠正措施，比如尝试不同的工具或者修正参数或者重新规划。错误与处理能力。
- 抗干扰能力，就是在环境中加入一些噪声或者误导性的信息来看，评估现在的成功率下降多少。

**3. 自主性(Autonomy & Alignment):**

- 一个需要人类干预的次数，需要人类干预次数越少，说明这智能更自主，需要人类帮助次数越少。
- 行为的可解释性，看Agent 的思考过程是否清晰和合乎逻辑。
- 计划的遵循程度，如果agent提前制定一个计划，看他在在执行过程当中，多大程度遵循了自己的计划。
