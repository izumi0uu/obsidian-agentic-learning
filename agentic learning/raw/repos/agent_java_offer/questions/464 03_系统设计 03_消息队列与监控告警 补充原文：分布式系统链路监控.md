### 分布式系统链路监控

链路监控的核心目标是三件事：
- 调用链路可视化；
- 链路性能监控；
- 链路故障排查。

常见实现思路是做一个统一的链路采集框架，让每一次服务调用都经过这个框架：
- 采集调用的起点、终点、耗时、异常信息；
- 把这些数据持久化；
- 再通过可视化界面展示整条调用链路、各节点耗时和异常点。

常见[[Trace|链路追踪]]体系包括：
- Google `Dapper`
- 阿里 `鹰眼`
- 大众点评 `CAT`
- Twitter `Zipkin`
- `SkyWalking`

面试里可以补一句：
**链路监控不是单纯记日志，而是要能把一次跨服务请求完整串起来，回答“这次请求经过了哪些服务、卡在了哪一跳、为什么失败”。**
