## 题目正文

### 2. 消费幂等如何做？

核心是为每条业务事件定义唯一幂等键（订单号、业务流水号等），消费前先查重。常见实现是去重表、唯一索引、redis分布式锁，确保重复消息不会重复生效。在线上先用 Redis SETNX 做高性能幂等，降低重复消费压力；对资金/发券等关键链路，再用数据库唯一键做最终兜底，避免 Redis 异常导致重复执行
