## 题目正文

### 重复消费与幂等

Kafka 消费端可能重复消费，本质原因是：
- 业务已经处理成功；
- 但 offset 还没来得及提交；
- 消费者重启后又会从旧 offset 再拉一次。

所以常见工程做法是：
- 关闭自动提交 offset；
- 处理成功后手动提交；
- 同时在业务层做幂等。

业务幂等常见做法：
- 用数据库唯一键防重；
- 或者用 Redis / 内存 Set 做去重判断。
