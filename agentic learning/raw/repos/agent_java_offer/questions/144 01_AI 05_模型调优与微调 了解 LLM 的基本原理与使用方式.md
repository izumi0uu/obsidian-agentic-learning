## 题目正文

### 2. 子问题：了解 LLM 的基本原理与使用方式

答：

它本质上是一个基于 [[Transformer]] 架构训练出来的大规模语言模型，Transformer 架构单encode是bert模型主要做相关性和意图识别, 单decode就是大语言生成模型. 核心能力是根据已有上下文去预测下一个 [[Token|token]]。因为它在海量语料上做过预训练，所以不只是会语法和表达，还学到了一定的知识、模式和推理能力。

如果从训练过程看，一般分三步。第一步是预训练，用大规模文本做 next-token prediction，获得通用语言能力；第二步是指令微调，让模型更适合问答、总结、抽取这类任务；第三步是后训练，比如 RLHF 或 DPO，让模型输出更符合人类偏好，也更安全、更稳定。

它的优势是泛化能力强，很多任务不需要单独训练模型，通过 prompt 就能先跑起来；但它的局限也很明显，比如会[[Hallucination|幻觉]]、可控性有限、成本和延迟都比较高。所以真正上线时，通常要配合 [[Prompt]] 约束、[[RAG]]、[[Tool Calling|工具调用]]、[[Evaluation|评估]]和监控一起用

追问：为什么同一个模型在不同系统提示词下表现差异会很大？

**LLM 不是执行固定代码，而是“在当前上下文条件下做下一个 token 的概率决策”**，系统提示词本身就是最强条件之一。
