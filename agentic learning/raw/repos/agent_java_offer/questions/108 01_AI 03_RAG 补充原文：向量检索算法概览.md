## 10. 补充原文：向量检索算法概览

### 2.4 向量检索

向量检索：随着深度学习算法发展，原本文本层面并不相似的数据，在高维向量空间中可能变得更加相关。将文本或图片转成向量后，在给定向量集合中，按某种度量方式检索出与查询向量最接近的 K 个向量。

向量检索算法概览：
- 暴力计算（brute-force）
  - 全量比较，找到与 query 向量最接近的 K 个结果。
  - 优点：结果准确，不存在误差。
  - 缺点：只适合小规模数据，无法支撑大规模在线检索。
- 基于哈希的方法：局部敏感哈希（LSH）
  - 利用哈希碰撞，把相似向量映射到同一个桶中。
  - 优点：避免和所有向量逐一比较，提升检索效率。
  - 缺点：数据量很大时，为了保证准确率，往往需要更多哈希函数。
- 基于倒排的方法：PQ / IVFPQ
  - 核心是对向量做压缩，把原始高维向量切分成多个子向量，对每段做量化编码。
  - 优点：显著降低内存占用，适合大规模[[Vector Database|向量库]]。
  - 缺点：量化会引入精度损失，召回率受聚类和编码参数影响。
- 基于图的方法：HNSW
  - 通过多层图结构建立“高速查询通道”，查询时像在图里做近邻跳转。
  - 优点：检索速度快、召回率高。
  - 缺点：内存占用大、建索引慢、删除节点不友好。
