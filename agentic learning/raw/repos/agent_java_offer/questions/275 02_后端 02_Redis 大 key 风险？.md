## 题目正文

### 10. 大 key 风险？

大key的本质问题: 一次命令处理数据太多，会放大 Redis 单线程模型的时延风险；治理关键是“提前发现、结构拆分、分页访问、异步删除”.
大 key 会导致单次操作阻塞事件循环、网络包过大、主从数据迁移和复制耗时长，还会造成集群槽位热点。治理方案: 1.拆分 key（核心）; 2.避免全量读写,用分页命令：HSCAN/SSCAN/ZSCAN、ZRANGE 分页，避免一次全拉; 3.删除用异步
