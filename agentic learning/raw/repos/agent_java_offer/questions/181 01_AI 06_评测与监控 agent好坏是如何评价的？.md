## 题目正文

### 1. 子问题：agent好坏是如何评价的？

**口述答案（约300字）**：
[[Agent]][[Evaluation|评估]]我会分“效果、效率、稳定性、安全性”四个维度。效果看任务完成率、答案正确率、引用准确率；效率看首token时延、端到端时延、单任务token和外部API成本；稳定性看[[Tool Calling|工具调用]]成功率、重试率、超时率、回环率；安全性看越权调用、敏感操作拦截率和审计完整性。评测方法上采用“离线集+线上流量”双轨：离线做可重复回归，线上看真实用户反馈与业务指标。最重要的是坏例回流机制，把失败case结构化记录并分类到Prompt、检索、工具、编排四层。面试里我会强调：评估不是单一准确率，而是以业务目标为核心的综合评分体系。
