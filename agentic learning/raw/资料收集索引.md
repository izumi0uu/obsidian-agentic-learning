#### 以后新 paper 的默认归类

- 支撑多个现有概念卡、能纠正常见误解、或是当前学习路线的地基证据：P0。
- 直接服务 Agent harness、workflow、trace、trajectory evaluation、tool security、RAG reliability 等当前工程判断：P1。
- 只在 memory、GUI、多 Agent、行业高风险应用等专题里才必要：P2。
- 主要解释模型训练、规模化、对齐或 reasoning model 背景：P3，除非当前问题正在学习 LLM / alignment。

小边界：P0/P1 论文读完后要进入 wiki 或 reviews；P2/P3 可以先停在 raw/source index，不急着制造新概念。

### 论文 source note 规则

新建 `raw/papers/` 笔记时优先使用 [[templates/论文]]。最低约束：

- frontmatter 必须有 `type: source`、`source_type: paper`、`title`、`url`、`topic`、`created`、`last_checked`、`freshness`、`status`、`related`。
- 如有 PDF 或抽取文本，使用 `pdf` 和 `extracted` 字段；本地文件放在 `raw/papers/assets/` 与 `raw/papers/extracted/`。
- 正文至少保留：`## 为什么收`、`## 一句话`、`## 需要我读的内容`、`## 论文主张`、`## 可以拆成概念卡`、`## 我的疑问`、`## 边界提醒`。
- `### 必读` 不是待读清单，而是必读证据提取区：每个必读块至少写 `位置`、`为什么必读`、`原文短摘`、`中文概括`、`支撑概念`、`证据边界`。
- 如果必读内容很长，只摘 1-3 句关键原话；其余用中文概括。原文短摘负责证据，中文概括负责学习，不能混在一起。
- raw paper note 只作为证据层；不要把它写成稳定概念卡。概念理解要写入 `wiki/concepts/`，并回链到论文 source note 小节或 PDF page / section。
- 没有精读到页码时，用 source note 小节锚点；不要假装有段落级证据。
