# 双链

## 一句话

双链是 Obsidian 里的页面链接方式，例如写成 `[[Agent]]`，用来把两个笔记页面连接成可互相发现的知识网络。

## 它解决什么问题

普通文件夹只能表达“这篇笔记放在哪里”。双链可以表达“这个概念和哪些概念有关”。

在这个 vault 里，双链让 LLM 和人都能沿着概念关系移动：

- 从 [[Agent]] 跳到 [[Agent Loop]]。
- 从 [[RAG]] 跳到 [[Retriever]]。
- 从 [[Oh My Codex (OMX)]] 跳到 [[Agent Harness]]。
- 从 source note 跳到 concept card。

## 它不是什么

双链不是分类系统的替代品。

它也不是链接越多越好。太多弱链接会让图谱变成噪音，真正有价值的是能解释“为什么相关”的链接。

双链也不能证明你已经理解了概念。只有能用自己的话解释，才算真正进入知识系统。

## 最小例子

```md
[[Agent]] 通常包含 [[Agent Loop]]、[[Tool Calling]]、[[Memory]] 和 [[Evaluation]]。
```

这句话会把五张卡连接起来，之后可以从任意一张卡反向发现其他卡。

## 常见误解和风险

- 误解：所有名词都应该做双链。
- 误解：图谱越密，知识越扎实。
- 风险：把临时词、路径、公式、图片名误写成双链，会制造不存在的页面。
- 风险：链接了页面但没有补内容，会留下“空承诺”。

## 边界细节

双链的价值在于表达“概念之间为什么相关”，不是替代目录、标签、frontmatter 或全文搜索。目录回答文件放在哪里，frontmatter 回答页面角色和状态，双链回答概念之间如何跳转。

在 LLM 维护的 wiki 里，双链还承担 grounding 作用：回答问题时优先沿现有概念卡和地图走，而不是每次从 raw 重新拼答案。但弱概念不应为了图谱好看而立即建页；先放 [[02 问题池]] 更安全。

## 现代性状态

双链对 Obsidian 来说是稳定基础功能；对这个 LLM Wiki 项目来说，它是当前个人知识管理实践，不是 Agent 领域前沿概念。本卡保持 seed-lite：重点是约束链接质量，而不是扩写成知识管理长文。

## 复习触发

- 双链、目录、tag、frontmatter 分别解决什么问题？
- 为什么不是所有名词都应该做成 `[[双链]]`？
- LLM 回答 wiki 查询时，双链怎样帮助它少依赖 raw 重拼？

## 使用规则

- 稳定概念用双链。
- 临时词先放普通文本。
- 外部文件路径不要写成双链。
- 弱概念先放进 [[02 问题池]]。
- 已经多次出现、并且能写出“它不是什么”的概念，再补成 wiki card。
