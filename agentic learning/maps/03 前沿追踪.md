# 03 前沿追踪

这里记录 Agent 前沿内容。每条必须有日期、来源、自己的判断和链接。

前沿概念全景：[[前沿主源清单]]

## 记录模板

```md
## YYYY-MM-DD 标题

来源：

一句话：

解决的旧问题：

可能的夸大：

我现在不懂的点：

链接到：
- 相关概念
```

## 追踪方向

- 长上下文和长期记忆
- 工具调用与计算机使用
- 多 Agent 协作
- 代码 Agent
- Agent 评估
- RAG 和知识库
- 安全、权限和人工确认
- Agent harness 和 eval harness
- GraphRAG、Agentic RAG、Agentic Retrieval
- Computer use 和 browser agent
- Agent protocol：MCP、A2A、ACP
- Agent observability：trace、replay、trajectory eval

## 判断标准

- 它是新能力，还是新包装？
- 它依赖模型进步，还是工程系统改进？
- 它在什么任务上有效？
- 它失败时会怎样失败？
- 它是否改变我对 [[Agent]]、[[Memory]]、[[Tool Calling]] 或 [[Evaluation]] 的理解？

## 仍需观察

大部分第一批前沿候选已经做成概念卡。现在只保留真正还不稳定的词：

- [[RAGGraph]]：目前更像项目名、RAG 工作流图或图知识库检索的混合说法，边界不如 [[GraphRAG]]、[[Agentic RAG]] 和 [[Agentic Retrieval]] 稳定。

后续遇到新词，不先建页；先在这里写一条观察，确认它是概念、协议、产品、项目名、工程能力还是风险模型。
