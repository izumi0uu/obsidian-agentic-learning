# 01 术语表

这里不是百科全书，而是我的 Agent 词感训练场。每个术语先用一句话解释；已经进入 wiki 的术语用双链，未建卡的基础词先保留短解释。

## 使用方法

读一个术语时只问三件事：

1. 它属于模型、知识、行动、记忆、评测、工程还是安全？
2. 它解决什么问题？
3. 它不是什么？

## A

- [[A2A]]：Agent 与 Agent 之间通信和互操作的协议方向。
- [[ACP]]：Agent Communication Protocol，关注 Agent、应用和人之间的标准化消息通信。
- [[AGENTS.md]]：给代码 Agent 读取的仓库级操作说明。
- [[Agent]]：围绕目标循环行动的系统，通常会观察状态、调用工具、更新记忆并修正计划。
- [[Agent State]]：Agent 当前任务运行中的目标、进度、中间结果、工具反馈和下一步依据。
- [[Agent Harness]]：包住模型、工具、状态、权限、运行环境、trace 和评测逻辑的执行外壳。
- [[Agent Lifecycle Hook]]：Agent runtime 在会话、工具调用前后、上下文压缩和停止等边界触发的本地或框架 handler。
- [[Agent Loop]]：Agent 的基本行动循环，常见形式是观察、思考、行动、反馈。
- [[Agent Workflow]]：把 Agent 任务组织成可控制步骤、分支、循环和交接的流程结构。
- [[Agentic RAG]]：Agent 主动决定何时检索、检索哪里、是否重查的 RAG 形态。
- [[Agentic Retrieval]]：检索层具备 query planning、子问题分解、多源选择和结果合并能力。
- [[Agent Framework]]：帮助开发者构建、编排、运行和观测 Agent 的软件框架。
- [[Approval Gate]]：高风险动作执行前要求人类确认的控制点。
- [[Audit Log]]：为安全、合规和复盘保存的 Agent 行动记录。

## B

- [[Benchmark]]：用于比较系统能力的一组任务、数据和评分方式。
- [[Browser Agent]]：能观察网页、点击、输入、滚动、提取信息并完成网页任务的 Agent。

## C

- [[Chunking]]：把长文档切成可检索片段的过程；切太大召回不准，切太小上下文不完整。
- [[Coding Agent]]：能读代码、改代码、跑测试、处理反馈的 Agent。
- [[Code Execution Sandbox]]：给 Agent 运行代码、命令或数据分析任务的隔离执行环境。
- [[Computer Use]]：让模型通过屏幕、鼠标、键盘、浏览器或终端操作计算机。
- context window：模型一次能看到的上下文范围；它不是长期记忆。
- [[Context Engineering]]：设计进入模型上下文的信息结构、顺序、预算和来源。
- [[Corrective RAG]]：检索后评估证据质量，不够好就改写查询、重检索或走补救路径。

## D

- [[Data Exfiltration]]：敏感数据被 Agent、工具、检索结果或模型输出带出安全边界的风险。
- [[Document Ingestion]]：把 PDF、网页、表格等原始资料转换成可检索、可引用知识单元的流程。
- [[Durable Execution]]：让长任务可以暂停、恢复、重试并保存运行状态的执行能力。

## E

- [[Embedding]]：把文本、图片等内容变成向量，让系统可以计算语义相似。
- [[Episodic Memory]]：保存过去事件、任务轨迹、操作结果和经验样例。
- [[Eval Harness]]：自动运行任务、记录过程、收集输出、打分并复现失败的评测外壳。
- [[Evaluation]]：用任务、样例、指标和人工判断来检查系统是否稳定有效。

## G

- [[GraphRAG]]：用知识图谱或图结构增强检索和回答。
- [[Guardrails]]：限制、检查或修正模型输入、输出和工具动作的安全/质量边界。
- [[GUI Grounding]]：模型把屏幕图像、控件、坐标和页面状态对应到可执行动作的能力。

## H

- [[Handoff]]：一个 Agent 把任务、上下文和控制权交给另一个 Agent 或人类的机制。
- [[Hermes Agent]]：Nous Research 的开源 self-improving Agent runtime，把记忆、skills、MCP、消息网关、工具执行和长期目标循环组合成具体 harness。
- [[Human-in-the-loop]]：在人类确认、修正、接管或审核下推进 Agent 工作流的机制。
- [[Hybrid Search]]：把向量语义检索和关键词/全文检索结合起来的检索方式。

## I

- [[Indirect Prompt Injection]]：模型读取网页、文档、邮件、检索片段或工具结果时，被隐藏恶意指令影响。

## L

- [[LangChain DeepAgents]]：LangChain 在 LangGraph 上封装的长任务 Agent harness，提供 planning、filesystem、subagents、memory 和 permissions 等脚手架。
- [[Least Privilege Tools]]：让 Agent 只拥有完成当前任务所需的最小工具、数据和动作权限。
- [[LLM]]：大语言模型，擅长根据上下文生成文本，但不天然拥有事实保证、长期记忆或真实行动能力。
- [[LLM Training Pipeline]]：让 LLM 通过预训练、后训练、偏好优化、推理强化、工具能力和评测闭环持续变强的训练流程。
- [[LLM-as-Judge]]：用另一个模型或同类模型作为评估器，对输出、引用、格式或过程评分。
- [[LLM Gateway]]：统一管理模型路由、fallback、限流、成本和日志的网关层。
- [[Long-term Memory]]：Agent 跨会话保存和重新使用信息的能力。

## M

- [[MCP]]：让 AI 应用以标准方式连接工具、数据源和上下文服务的协议。
- [[MCP Registry]]：用于发现、发布和分发 MCP server 的注册表基础设施。
- [[Memory]]：Agent 保存和使用过去信息的机制。
- [[Memory Reflection]]：从历史对话、任务轨迹或事件记忆中总结长期记忆的过程。

## N

- [[Neo4j]]：图数据库和 GraphRAG 工程生态，可把实体、关系、chunk、向量索引、全文索引和 Cypher 查询组合进 RAG。

## O

- [[Observation]]：Agent 动作后由工具、环境、测试或人类返回的反馈，会被写回上下文或 state，影响下一步决策。
- [[Observability]]：让 Agent 系统的输入、输出、工具调用、延迟、成本、错误和质量信号可观察。
- [[Obsidian + LLM Wiki]]：由双链笔记组织、由 LLM 辅助提问和整理的个人知识系统。
- [[Oh My Codex (OMX)]]：OpenAI Codex CLI 的多 Agent 编排和运行时增强层。
- [[OMX $ 指令]]：在 Codex 会话里触发 OMX skill / workflow 的入口，用来切换澄清、计划、执行、评审、研究或 goal-gated 工作方式。
- [[OpenTelemetry GenAI]]：OpenTelemetry 为生成式 AI 调用、token、模型和工具事件定义的观测语义约定方向。

## P

- [[Patch Validation]]：用测试、lint、typecheck 或人工审查验证代码修改。
- [[Plan-and-Solve Prompting]]：先让模型写计划，再让模型按计划求解的 zero-shot CoT prompting 方法。
- [[Planning]]：把目标拆成步骤，并在执行过程中调整步骤。
- [[Policy Engine]]：根据规则、上下文和风险等级决定 Agent 能否调用工具、访问数据或执行动作的控制层。
- prompt：给模型的输入指令和上下文；它能影响输出，但不能单独保证事实正确或安全。
- [[Prompt Injection]]：攻击者用输入内容诱导模型违背原始任务、系统指令或安全边界。

## R

- [[RAG]]：Retrieval-Augmented Generation，先检索外部资料，再让模型基于资料回答。
- [[RAG Evaluation]]：评估检索、上下文、引用和最终回答质量的一组方法。
- [[RAGGraph]]：不稳定说法，可能指 RAG 工作流图，也可能指图知识库检索；先不要当作成熟概念。
- [[Reasoning Trace]]：模型显式写出的推理过程记录。
- [[ReAct]]：把 reasoning 和 acting 交替组织起来的 Agent 方法。
- [[Reflexion]]：Agent 执行后根据反馈生成反思文本，并把它作为经验记忆改进下一轮行动的机制。
- [[Replay]]：用保存的输入、工具结果、检索结果或环境快照重放 Agent 执行过程。
- [[Repo Context]]：代码库结构、规范、依赖、测试和历史上下文。
- [[Retriever]]：从外部知识库中找出相关材料的组件。
- [[Reranking]]：在初步检索后用更精细模型或规则重新排序候选上下文。

## S

- [[Sandbox Workspace]]：给 Agent 执行命令、读写文件或操作 GUI 的隔离工作环境。
- [[Self-RAG]]：让模型自适应判断是否检索、如何生成、证据是否支持答案的一类 RAG 方法。
- [[Semantic Memory]]：保存相对稳定的事实、偏好、概念和关系。

## T

- [[Task Success Rate]]：端到端任务是否完成的比例。
- [[TF-IDF]]：把文本表示成词项权重稀疏向量，用词频和逆文档频率衡量词对文档的区分度。
- [[Top-K]]：按分数排序后取前 K 个候选；在 RAG 里常指检索前 K 个 chunk，在生成解码里则指 token 采样候选。
- token：模型处理文本的基本单位，不完全等于字或词。
- [[Tool Calling]]：模型输出结构化工具调用请求；schema 是工具说明书和参数契约，真正执行仍在外部程序。
- [[Tool Permissioning]]：给 Agent 工具和工具动作设置权限、范围、审批和风险等级。
- [[Tool Poisoning]]：工具描述、schema、元数据、返回值或供应链被污染，诱导 Agent 错误调用工具。
- [[Tool Registry]]：管理 Agent 可用工具名称、schema、描述、版本、来源、权限和状态的目录。
- [[Tool Use]]：模型或 Agent 使用外部工具完成自身不能直接完成的任务。
- [[Trace]]：记录 Agent 每一步输入、输出、工具调用、工具结果、状态变化和最终结果。
- [[Trajectory]]：一次 Agent 任务实际走过的行动路径；[[Trace]] 是对它的记录，[[Reasoning Trace]] 是其中的推理文字部分。
- [[Trajectory Evaluation]]：评价 Agent 的行动轨迹，而不是只评价最终答案。

## V

- [[Vector Database]]：为 embedding 向量检索设计的数据库或索引系统。

## Z

- [[Zero-shot CoT]]：不提供示例，只用“让我们一步步思考”这类提示触发模型生成中间推理步骤的 prompting 方法。

## 容易混淆

- [[LLM]] 不是 [[Agent]]：LLM 生成内容，Agent 组织行动。
- [[Tool Calling]] 不是 Agent：工具调用只是 Agent 可用的一种能力。
- [[Oh My Codex (OMX)]] 不是新模型：它是 Codex 外面的 workflow、harness、team runtime 和状态管理。
- [[OMX $ 指令]] 不是 shell 命令：`$ralph` 是会话里的 skill 触发入口，`omx ...` 才是终端 CLI；两者可能配合，但不在同一层。
- [[LangChain DeepAgents]] 不是 LangGraph 本身：它是建在 LangGraph 上的 harness，LangGraph 才是更底层的状态图编排和运行时。
- [[RAG]] 不是 [[Memory]]：RAG 是检索知识，记忆还包括状态、偏好、历史和任务过程。
- [[Neo4j]] 不是 [[GraphRAG]]：Neo4j 是实现工具和图数据库生态，GraphRAG 是图增强检索的方法或模式。
- context window 不是 [[Long-term Memory]]：上下文会随对话或任务边界变化，长期记忆需要外部保存和检索。
- [[Trace]] 不是 [[Evaluation]]：trace 记录发生了什么，evaluation 判断好不好。
- [[Trajectory]] 不是 [[Trace]]：trajectory 是任务路径本身，trace 是系统保存下来的观测记录。
- [[Trajectory]] 不是 [[Reasoning Trace]]：trajectory 包括动作、观察、工具结果和状态变化；reasoning trace 只指模型显式推理文字。
- [[Zero-shot CoT]] 不是 [[ReAct]]：它只有文本推理链，没有 Action/Observation 的外部循环。
- [[MCP]] 不是 [[A2A]]：MCP 偏 Agent 连接工具和数据源，A2A 偏 Agent 与 Agent 通信。
- [[Prompt Injection]] 不是“prompt 写得不好”：它是外部内容进入模型控制边界后的安全问题。
- [[Approval Gate]] 不是每一步都问人：它只应该卡住高风险、不可逆或越权动作。
