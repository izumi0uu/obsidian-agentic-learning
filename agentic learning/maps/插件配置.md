# 插件配置

当前策略：先用 Obsidian 核心能力 + Web Clipper + Dataview。AI 问答等笔记量上来再接。

Codex 侧已经有一个本地 skill：

```text
~/.codex/skills/obsidian-llm-wiki
```

它会读取本项目的 `AGENTS.md`、[[LLM Wiki 工作流]] 和 [[字段规范]]，再执行 ingest/query/lint。

## 必装

### Obsidian Web Clipper

用途：把网页和元数据保存为 Markdown。

建议默认保存到：

```text
raw/inbox/
```

建议使用模板：

```text
templates/网页剪藏.md
```

安装边界：Web Clipper 是浏览器扩展，不是 vault 内的社区插件。vault 端已经准备好 `raw/inbox/` 和 `templates/网页剪藏.md`，浏览器扩展本身需要在浏览器扩展商店或 Obsidian 官方页面安装。

### Dataview

用途：把 frontmatter 字段变成可查询索引。

安装后，`index.md` 和 [[Agent 知识地图]] 里的 dataview 代码块会自动显示表格。

当前状态：插件文件已安装；如果 Obsidian 里表格没有显示，到设置里确认 Community plugins 是否启用了 Dataview。

### Templater

用途：在模板里自动填日期、标题和元数据。

当前状态：插件文件已安装；如果新建模板没有自动日期，到设置里确认 Community plugins 是否启用了 Templater。模板目录为：

```text
templates/
```

当前策略：先不开 folder template 和 system command。等模板稳定后再增加自动化，避免新手期被自动规则干扰。

## 主题和字体

### Minimal

当前状态：已安装并设为主题。

### 字体

当前已启用 CSS snippet：

## 暂缓

### Copilot + Ollama 或 LM Studio

等至少有 30-50 张概念卡后再配置。否则 AI 问答只是在问一堆未经整理的剪藏资料。

这和 Codex skill 不冲突：Codex skill 负责维护 vault，Copilot/Ollama 未来负责在 Obsidian 内问答。

## status 约定

- `inbox`：刚收集，还没处理。
- `seed`：已经有初步理解。
- `growing`：正在补例子、边界和链接。
- `mature`：相对稳定，可以复习或引用。
- `review`：需要重新检查或更新。
