# 00 学习路线

这条路线默认我从 0 开始。每一阶段只追求一个核心变化：从“听过这个词”到“能用自己的话解释它的边界”。

## 总原则

- 每天只保底 30-60 分钟。
- 每天只产出一个小东西：一句定义、一张卡、一个问题、一个例子或一个边界。
- raw 只存来源，wiki 存理解，daily 存当天推进。
- 概念没法用自己的话解释，就还没有学会。

## 辅助教材

- [[Hello-Agents Repo]]：中文系统化入门和实践教材。适合补路线和案例，不替代官方文档、论文和协议规范。

## 阶段完成标准

### 阶段 1：基础词感

- 我能解释 [[LLM]] 为什么不是数据库。
- 我能解释 context window 为什么不是 [[Long-term Memory]]。
- 我能解释 [[Agent]] 和普通聊天机器人的最小区别。

### 阶段 2：系统结构

- 我能画出一个 observe -> think -> act -> observe 的 [[Agent Loop]]。
- 我能说出 [[Tool Calling]]、[[Memory]]、[[Planning]]、[[Evaluation]] 分别解决哪层问题。
- 我能给一个 Agent 失败案例定位到具体环节。

### 阶段 3：知识和前沿

- 我能解释 [[RAG]]、[[GraphRAG]]、[[Agentic RAG]]、[[Agentic Retrieval]] 的边界。
- 我能判断一个新词是概念、协议、项目名、产品名还是工程能力。
- 我能从主源里拆出 1-3 张概念卡，而不是只复制摘要。

### 阶段 4：安全和自维护

- 我能解释 [[Prompt Injection]]、[[Tool Poisoning]]、[[Approval Gate]] 和 [[Policy Engine]] 的关系。
- 我能用 [[01 术语表]] 找到概念入口。
- 我能用 daily 推动 raw -> wiki -> map 的学习循环。

## 每日最小动作

每天只要完成以下任意一个，就算推进：

- 新建一张概念卡。
- 回答 [[02 问题池]] 里的一个问题。
- 给一张旧卡补一个“它不是什么”或“最小例子”。
- 把一条前沿信息写进 [[03 前沿追踪]]。
- 做一个 15 分钟小实验并写进 [[实验记录索引]]。

## 小边界

这 30 天不是为了“读完所有资料”。它是为了训练一个习惯：每次接触新概念，都能问出它的来源、用途、边界、风险和最小例子。
